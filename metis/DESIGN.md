# metis CLI design decisions

Design session log — decisions made so future sessions don't re-litigate them.

---

## v0.1 — Initial email redesign (June 2026)

**Chosen direction:** "Lenny's Newsletter energy" — clean, editorial, no visual noise.

### Email design (DigestHeader.tsx)
- Metis wordmark: 8×8px black square + "Metis" text + date right-aligned
- Greeting line: time-of-day + apply count (generated by `render.py:_make_greeting()`)
- Title: "Metis Digest" (fallback when no candidate name set; replaced by personalized greeting otherwise)
- Stat tiles: 3 filled colored tiles (Evaluated / Solid Match / Moderate Match)
- Legend row with colored dots
- Footer: "Metis · powered by <provider> · N roles evaluated"

**Decided NOT to add:**
- Unsubscribe link — this is a self-addressed CLI tool; `metis schedule --remove` is the equivalent
- Dynamic email subject — user prefers stable subject for grep/threading

## v0.2 — Init wizard reorder (June 2026)

**New wizard flow:** who you are → what you want → what you won't accept

Sections:
1. Resume + LinkedIn (who you are)
2. Target roles + career direction (what you want)
3. Constraints + deal-breakers (what you won't accept)
4. Review and save

**Decided:**
- LinkedIn step: requires PDF export (not URL) — disclosed upfront before confirm
- Overrides/edge cases: skipped in main wizard flow, available only in review menu

## v0.3 — Theme system + design overhaul (June 2026)

### Single source of truth: `theme.py`
All colors, questionary style, InquirerPy style, and print helpers live in `metis/theme.py`.
Nothing else defines colors.

### Color hierarchy (most → least prominent)
```
User typed input      → terminal default fg (no color override = always brightest)
Question label        → THEME['label'] bold  (#c8c8c5 dark / #2d2d2d light)
Option body text      → THEME['muted']
Hint / instruction    → THEME['dim'] italic
Structural chrome     → THEME['dim']
Active symbol / rule  → THEME['accent']  (#4a9edd dark / #1d5aad light)
Confirmed steps       → THEME['success'] (green — semantic only, ✓ lines)
Warnings              → THEME['warning'] (amber — semantic only)
Errors                → THEME['error']   (red — semantic only)
```

**Key inversion from original:** decorative elements go DOWN in brightness, user input goes UP.
The `answer` token in questionary is `"bold"` (no color) so terminal default = pure bright.

### Spacing golden rule (Gemini consensus, adopted)
- WITHIN a question block: 0 blank lines. Label → hint → input are one visual unit.
- BETWEEN question blocks: exactly 1 blank line.
- The `_ask*` helpers enforce this: each starts with `console.print("\n  [bold]{label}")`.

### Prompt pointer
Fixed-width prefix `"  › "` (2 spaces + › + 1 space) on every prompt type via InquirerPy
`message="  › "`. This ensures horizontal alignment across text inputs, selects, and checkboxes.

### Section headers
Replaced dark navy Panel with Rich `Rule` (horizontal line + title at left edge):
```
── 2 of 4  What you want  — roles, direction  ─────────────────────────
```
Less visually intrusive than a filled background panel. Callers manage blank lines around it.

### Welcome panel
Rich Panel with `ROUNDED` box (╭─╮), accent border, `welcome_bg` background.
Dark: `#111116` (warm very-dark). Light: `#f4f4f8` (cool off-white).

### Terminal width floor
Min 80 cols. Below that: print warning + `sys.exit(1)`. No reflow attempt below threshold.

## v0.4 — Profile display + quick edit (June 2026)

### Profile section order (both `_show_profile` and quick edit menu)
```
── Identity        name, location, work preference (remote/local/relocate/flexible)
── Target          roles, level, track, direction, industry targets, stage preference
── Constraints     deal-breakers, salary floor
── Strengths       differentiators, proof points
── Green flags     boost signals, company types drawn to
── Gaps            yellow flags, red flags (surfaced by the configured LLM, user can correct)
── Scoring config  apply threshold (default 75), consider threshold (default 55)
── Scoring overrides  free-text edge cases injected into the scoring prompt
```

### Quick edit principle
Every section in `_show_profile` has a 1:1 corresponding option in the review loop menu.
Previously: "Target" and "Constraints" existed but "Identity", "Gaps", "Scoring config" did not.
Now all 8 sections are independently editable.

## Prompt library migration

Original: `questionary` throughout.
Current: `InquirerPy` for all new `_ask*` helpers in `init_cmd.py`.

The `_ask*` helper family (`_ask`, `_ask_select`, `_ask_checkbox`, `_ask_confirm`, `_ask_filepath`)
wraps InquirerPy with the spacing + pointer rules baked in. Use these for any new prompts.
The review loop handlers may still use raw `questionary` calls — migrate as needed.

`QUESTIONARY_STYLE` (legacy) and `INQUIRER_STYLE` (current) both live in `theme.py`.
Both adapt to dark/light terminal mode.
